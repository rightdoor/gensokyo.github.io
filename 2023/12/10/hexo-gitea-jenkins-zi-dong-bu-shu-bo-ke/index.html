Hexo+Gitea+Jenkins自动部署博客

2023-12-10 18:33:18
实用笔记
1.1k 词
  • CI/CD
  • Gitea
  • Jenkins
  • docker
  • hexo

hexo不能一边推送一边自动将网站部署到nginx，所以使用Docker安装的Gitea和Jenkins组合来自动部署静态页面。

自动化部署Hexo博客

1、Docker安装Gitea和Jenkins

在Jenkins部署时记得映射一个目录指向nginx的静态页面文件夹。

在Gitea部署完成后修改app.ini文件。

2、Gitea

1、设置好Gitea令牌，保存好。

2、新建仓库，设置好Web钩子。

创建选择Gitea，将url设置为以下格式：

http://192.168.31.101:18580/generic-webhook-trigger/invoke?token=[自定义token]

3、Jenkins

进入页面Dashboard->系统管理->插件管理->available plugins

安装插件NodeJS Plugin和Generic Webhook Trigger Plugin和Gitea Plugin。

进入页面Dashboard->系统管理->全局工具配置->NodeJS安装

选择好node版本，Global npm packages to install填写hexo，保存。

进入页面Dashboard->系统管理->凭据

点击System，点击Add domain，创建好后点击Add Credentials。

类型选择Gitea Personal Access Token，范围选择全局，填写好Token保存。

返回首页新建任务，选择构建一个自由风格的软件项目。

源码管理填写好Gitea仓库地址，Credentials选择更改创建好的凭据，填写好分支。

构建触发器选择Generic Webhook Trigger，只填写Token，内容为前面Web钩子自定义的Token。

构建环境选择Provide Node & npm bin/ folder to PATH，选择刚刚设置好的node环境。

Build Steps选择执行 shell。

参考：

自定义好shell脚本后保存，点击立即构建。

结束

当我们推送hexo项目到Gitea仓库时，Jenkins也会自动构建然后把静态文件移动到Nginx网站目录。
